# Communication Module

This directory contains the TCP server and the application protocol used by the OTA system.

The communication stack is split into small layers:

- `tcp_server.c`: low-level TCP socket setup, accept, receive, and send.
- `tcp_frame_server.c`: binary frame encode/decode, escaping, sequence byte, and CRC16 validation.
- `tcp_protocol_server.c`: command dispatch for `PING`, `OTA_START`, `OTA_DATA`, and `OTA_END`.
- `tcp_server_task.c`: scheduler-friendly task that receives bytes, accumulates pending data, decodes frames, and calls the protocol handler.

## Frame Format

```text
[SOF][LEN(2)][SEQ][CMD][DATA...][CRC(2)]
```

- `SOF`: start-of-frame byte.
- `LEN`: payload length.
- `SEQ`: sequence number generated by the sender.
- `CMD`: protocol command.
- `DATA`: command payload.
- `CRC`: CRC16 over frame metadata and payload.

Special bytes are escaped by the frame layer so payload data can safely contain frame control bytes.

## Protocol Commands

```text
0x01  CMD_OTA_START
0x02  CMD_OTA_DATA
0x03  CMD_OTA_END
0x10  CMD_PING
0x80  CMD_ACK
0x81  CMD_NACK
```

The server responds with `ACK` when a command is processed successfully and `NACK` when validation or handling fails.

## OTA Flow

```text
OTA_START -> OTA_DATA... -> OTA_END
```

`OTA_START` carries the firmware size and expected CRC32. `OTA_DATA` carries firmware chunks. `OTA_END` triggers final sync, size validation, CRC validation, binary swap, and restart.

The PC-side client lives in `tcp_client/`.
